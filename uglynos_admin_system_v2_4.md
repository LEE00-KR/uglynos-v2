## 1. 개요

### 1.1 시스템 목적

어글리노스 게임의 모든 콘텐츠를 관리하기 위한 어드민 시스템입니다. 페트, 스킬, 스테이지, 아이템 등 게임 내 모든 데이터를 등록, 수정, 삭제할 수 있으며, 각 항목별로 코드 및 프롬프트를 자동 생성하는 기능을 제공합니다.

### 1.2 주요 기능

- **데이터 관리**: 페트, 스킬, 아이템, 스테이지 등 게임 데이터 CRUD
- **코드 생성**: 등록된 데이터 기반 개발용 코드 자동 생성 (개별/다중)
- **프롬프트 생성**: AI 활용을 위한 프롬프트 자동 생성
- **시뮬레이션**: 페트 성장률 등급별 미리보기
- **미리보기**: 스테이지 페트 배치 등 시각적 확인

### 1.3 문서 연동 안내

> 💡 **전투 시스템 연동**  
> 이 문서의 일부 항목은 `uglynos_battle_system_v1.2.md` 문서와 연동됩니다.  
> - **어드민 시스템**: 데이터 등록/관리 UI 및 항목 정의
> - **전투 시스템**: 등록된 데이터의 실제 적용 로직 및 공식
> 
> 연동 항목에는 `→ 전투 시스템 연동` 표시가 있습니다.

## 2. 페트 관리

### 2.1 기본 정보

| 항목 | 상세 | 비고 |
|------|------|------|
| 페트명 | 텍스트 입력 | 필수 |

## 3. 성격 관리 (성격DB)

### 3.1 관리 항목

| 항목 | 상세 | 비고 |
|------|------|------|
| 성격명 | 텍스트 입력 | 필수 |
| 상승 스탯 | HP/ATK/AFF/DEF/SPD 선택 | 단일 선택 |
| 상승률 | % 입력 | 기본 +10% |
| 하락 스탯 | HP/ATK/AFF/DEF/SPD 선택 | 단일 선택 |
| 하락률 | % 입력 | 기본 -10% |
| 출현 가중치 | 숫자 입력 | 높을수록 자주 등장 |
| 활성화 여부 | ON/OFF 토글 | 비활성 시 미사용 |

### 3.2 성격 예시

| 성격명 | 상승 스탯 | 상승률 | 하락 스탯 | 하락률 |
|--------|----------|--------|----------|--------|
| 용감함 | ATK | +10% | DEF | -10% |
| 신중함 | DEF | +10% | SPD | -10% |
| 민첩함 | SPD | +10% | HP | -10% |
| 건강함 | HP | +10% | ATK | -10% |

## 4. 스킬 관리

### 4.1 기본 정보

| 항목 | 상세 | 비고 |
|------|------|------|
| 스킬명 | 텍스트 입력 | 필수 |
| 타입 | 물리 / 속성 | 단일 선택 |
| 속성 | 지/수/화/풍 | 물리 선택 시 비활성 |
| **공격 범위** | 근접 / 원거리 | **v2.4 추가** → 전투 시스템 연동 |
| 스킬값 | 데미지 배율 | 숫자 입력 |
| 명중률 | % 입력 | 기본 100% |
| 크리티컬 보정 | 배율 입력 | 기본 1.0 |
| 쿨타임 | 턴 수 입력 | 0 = 쿨타임 없음 |

### 4.2 공격 범위 설정 ← **v2.4 신규**

> → **전투 시스템 연동**: 반격 가능 여부 판정에 사용

| 범위 | 설명 | 반격 가능 여부 |
|------|------|:-------------:|
| 근접 | 직접 접촉 공격 | ✅ 반격 가능 |
| 원거리 | 투사체/원거리 공격 | ❌ 반격 불가 |

```
┌─────────────────────────────────────────────────────┐
│ 공격 범위:                                          │
│ ○ 근접 (반격 가능)                                  │
│ ○ 원거리 (반격 불가)                                │
└─────────────────────────────────────────────────────┘
```

### 4.3 효과 설정

| 항목 | 상세 | 비고 |
|------|------|------|
| 스탯 변동률 | HP/ATK/DEF/SPD 각각 % | 기본 100% |
| 대상 | 적 단일/전체, 아군 단일/전체, 자신 | 선택 |
| 상태이상/특수효과 | 상태이상DB에서 선택 | 발동 확률 입력 |

### 4.4 코드 생성 기능

## 6. 훈련소 관리

### 6.1 개요

훈련소는 유저가 페트에게 새로운 스킬을 가르치는 시스템입니다. 어드민에서는 훈련소에서 배울 수 있는 스킬과 비용을 등록합니다.

### 6.2 관리 항목

| 항목 | 상세 | 비고 |
|------|------|------|
| 스킬 선택 | 스킬DB에서 선택 | 필수 |
| 습득 비용 | 재화 종류 + 수량 | 필수 |
| 코드/프롬프트 생성 | 개별 생성 | 다중 생성 없음 |

### 6.3 훈련소 등록 UI

```
┌─────────────────────────────────────────────────────┐
│ 스킬: [스킬DB 드롭다운 ▼]                           │
│ 습득 비용: [재화 선택 ▼] [수량 입력: 1000]          │
│                                                     │
│ [코드 생성] [프롬프트 생성] [저장]                  │
└─────────────────────────────────────────────────────┘
```

### 6.4 훈련소 목록

├────────────┼─────────────┼─────────────────────────┤
│ 화염구     │ 1,000 스톤  │ [수정] [삭제] [코드]    │

### 6.5 훈련소 유저 플로우

```
① [훈련소] 입장
    │
    ▼
② [스킬 선택]
    │ - 훈련소에 등록된 스킬 목록 표시
    │ - 스킬별 비용 표시
    │
    ▼
③ [스킬 정보 노출]
    │ - 스킬 상세 정보
    │ - 습득 비용
    │
    ▼
④ [스킬 배울 대상 지정]
    │ - 유저 보유 페트 목록 표시 (유저 데이터에서 조회)
    │ - ⭐ 페트DB의 '습득가능스킬'에 해당 스킬이 등록된 페트만 활성화
    │ - 등록되지 않은 페트는 비활성(회색) 처리
    │
    ▼
⑤ [지정 후 스킬 목록 노출]
    │
    ├── 스킬 창 비어있음 → 즉시 습득 (1번 스킬창에 배치)
    │
    └── 스킬 창 꽉 참 → [교체할 스킬 선택]
                              │
                              ▼
                         [선택 스킬과 교체]
    │
    ▼
⑥ [스킬 습득 완료]
    │ - 재화 차감
    │ - 습득 완료 연출
    │
    ▼
⑦ [훈련소 메인으로 복귀]
```

### 6.6 훈련소 활성화 판단 로직

페트가 훈련소에서 특정 스킬을 배울 수 있는지는 **페트DB의 '습득가능스킬'** 등록 여부로 자동 판단됩니다.

```
유저가 훈련소에서 "화염구" 스킬 선택
    │
    ▼
유저 보유 페트 목록 조회
    │
    ├── 파이어렉스 → 페트DB 확인 → 습득가능스킬에 "화염구" 있음 → ✅ 활성화
    ├── 아쿠아돈   → 페트DB 확인 → 습득가능스킬에 "화염구" 없음 → ❌ 비활성화
    ├── 볼카노스   → 페트DB 확인 → 습득가능스킬에 "화염구" 있음 → ✅ 활성화
    └── 윈드랩터   → 페트DB 확인 → 습득가능스킬에 "화염구" 없음 → ❌ 비활성화
```

| **1회** | 계정당 단 1회만 구매 가능 |
